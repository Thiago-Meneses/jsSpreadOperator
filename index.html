Spread Operator

The spread operator let's you copy JavaScript Objects safely

Instead of changing the original Object, the spread operator creates a new instance of this Object

Always use the spread operator whenever you're trying to copy a reference type

Example:

Do NOT do this:

                                    
const myPerson = {
    name: "John",
    name: 32
}

let myPerson2 = myPerson

Do this instead:

                                    
const myPerson = {
    name: "John",
    name: 32
}

let myPerson2 = { ... myPerson }

==

The == is a Comparison Operator

This operator will transform the operands before doing a comparison.

Sometimes two operands can have same values but different types, consequently the comparison would return false

This can cause confusion in a situation where the programmer expects one boolean to be returned but doesn't

Example:

The following code compares myString and myNum

Both are assigned 5 as their value

However, their types are not the same

                                        
let myString = "5"
let myNum = 5

if ( myString == myNum )
    console.log(true);
else
    console.log(false);

Console:

                                        
true

Even if their types were different, true was returned

The == comparison operator converted the operands so that they had the same type

Even if we didn't care about types, and just wanted to know if both operands had the same value, this forceful behaviour is not ideal.

This operator has complete disregard for the types, and just expects to return true as much as possible

Although this behaviour seems to be helpful, it is not.

Converting types can affect results

Returning true should not be the goal, but returning the correct boolean should.

The act of converting types is essentially altering the initial comparison

Comparisons with different types should be primarily handled by the programmer.

If a certain comparison returns an unexpected boolean, the fault is probably in the reasoning of the programmer behind it.

===

The === is also a Comparison Operator

This operator does not transform the types of operands before a comparison.

This operator compares operands with their actual types.

Example:

This is the exact same example from the == operator

                                        
let myString = "5"
let myNum = 5

if ( myString === myNum )
    console.log(true);
else
    console.log(false);

Console:

                                        
false

This time the operator did not transform the types of the operands
